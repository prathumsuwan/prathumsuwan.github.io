ความเหมือนและความแตกต่างในการเขียนโปรแกรมแบบ OOP ด้วยภาษา Java, C++, Python

ความเหมือน

  • การประกาศและใช้งานคลาสและวัตถุ - ทั้งสามภาษารองรับการประกาศคลาสและสร้างออบเจ็กต์จากคลาส
  • Inheritance - ภาษาทั้งสามรองรับการสืบทอดจากคลาสหลัก (parent class) ไปยังคลาสย่อย (subclass) ซึ่งทำให้สามารถใช้คุณสมบัติของคลาสหลักในคลาสย่อยได้
  • Polymorphism - ทั้ง Java, C++ และ Python รองรับการพ้องรูป โดยสามารถใช้วิธีการเดียวกันในคลาสย่อยที่มีการปรับเปลี่ยนพฤติกรรมเฉพาะของแต่ละออบเจ็กต์
    • Java: 
      Animal dog = new Dog("Buddy");
      dog.makeSound();  // Output: Bark
    • C++: 
      Animal* dog = new Dog("Buddy");
      dog->makeSound();  // Output: Bark
    • Python: 
      dog = Dog("Buddy")
      dog.make_sound()  # Output: Bark
  • Encapsulation - ทั้งสามภาษารองรับการควบคุมการเข้าถึงข้อมูลและเมธอดภายในคลาส โดยใช้ access modifiers เพื่อควบคุมว่าใครสามารถเข้าถึงข้อมูลได้
    • Java: ใช้ private, protected, และ public
    • C++: ใช้ private, protected, และ public
    • Python: ใช้ underscore _ และ double underscore __
  • Constructor - ทั้งสามภาษารองรับการใช้คอนสตรัคเตอร์ในการสร้างออบเจ็กต์
  • Static Members - ทั้งสามภาษารองรับสมาชิกที่เป็น static

ความแตกต่าง

  • การประกาศและใช้งานคลาส
    • Java: ต้องกำหนดชนิดของตัวแปรอย่างชัดเจนและมีโครงสร้างเคร่งครัด
    • C++: รองรับ pointer และ reference และสามารถสืบทอดหลายคลาสได้
    • Python: ไม่ต้องระบุชนิดของตัวแปรล่วงหน้าและมีโครงสร้างยืดหยุ่น
  • Inheritance
    • Java: รองรับการสืบทอดจากคลาสเดียว แต่ใช้ interfaces แทนการสืบทอดหลายคลาส
    • C++: รองรับการสืบทอดหลายคลาส
    • Python: รองรับการสืบทอดหลายคลาสเช่นเดียวกับ C++
  • Polymorphism
    • Java: ใช้ method overriding และ interfaces หรือ abstract classes
    • C++: ใช้ pointers หรือ references กับ virtual functions
    • Python: ใช้ duck typing ซึ่งไม่ต้องระบุชนิดข้อมูลอย่างชัดเจน
